# v0.5 — Obsidian Sync Team Vaults

## Status

Shipped in v0.5, experimental. For teams already paying for Obsidian Sync who
want shared memory without any git plumbing. The recommended team transport
remains git (canonical, auditable); this mode trades those guarantees for
zero-plumbing convenience, and says so everywhere it matters.

## Problem

v0.2 team vaults require a git repository, a hosting account, and clones.
Teams that already share an Obsidian vault through **Obsidian Sync** have a
working multi-machine file transport — they just can't use it as a Context
Vault team vault, because the CLI assumes `sync: "git"`.

## Design

A third vault sync mode: `sync: "obsidian"`. The CLI performs **no sync
operations at all** for such vaults — Obsidian Sync moves the files. Context
Vault reads and writes the local synced folder like a personal vault, while
keeping the *team* semantics:

- **Attribution** — identity required, records stamped `author`/`agent`
  (client-asserted, as everywhere).
- **Disputes, topics, repo facets, withdraw, auto mode** — all work
  unchanged: they are schema features, not transport features.
- **Two-team cap** — obsidian vaults count toward the same limit as git
  vaults.
- **`init-obsidian-team --path <synced-folder>`** — the user creates/joins
  the shared vault in the Obsidian Sync UI first; the command validates the
  folder, ensures the layout, writes `ONBOARDING-OBSIDIAN.md`, creates the
  person note, and registers the vault in config. No clone, no driver, no CI.

## Honest degradations (vs. git team vaults)

| Capability | Git vault | Obsidian vault |
|---|---|---|
| Freshness info | last-synced / unpushed counts | **unknown** — Sync is opaque to the CLI; briefs say so |
| Conflict handling | merge driver + `merge_status` marks + quarantine | Obsidian Sync's own merge (may blend silently); mitigated by append-only records and rare mutable-note edits |
| History / audit | git log per record | none — Sync version history only, per file, in the Obsidian UI |
| `retract --remove-from-current-tree` | receipt-gated revert | **unavailable** — `withdraw` is the only correction |
| CI validation | vendored workflow | none — `doctor` checks the folder locally |
| Ledger push state | pushed / pending-sync | `obsidian-sync` (managed externally) |

Record collision safety survives without git: filenames carry random
suffixes and files are exclusively created, so two teammates writing at once
produce different files that Sync propagates without conflict.

## Out of scope

Mixing transports for one vault; making Obsidian Sync state visible to the
CLI (no API exists); replicating the merge driver (Sync applies its own
merging before the CLI ever sees files).
